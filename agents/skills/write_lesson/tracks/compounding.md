# Track Calibration: Compounding

## Core Learner Shift

Move from repeatable systems into systems that get stronger because prior work, decisions, and artifacts become trusted future leverage.

## What Lessons Should Optimize For

- Memory quality
- Retrieval trust
- Reusable artifacts
- Continuity across sessions
- Curation over accumulation

## Authoring Direction

- Teach what should be remembered and what should be discarded
- Show how prior work becomes usable in future work
- Emphasize retrieval, trust, and context hygiene
- Make cross-session continuity explicit
- Use examples where memory changes the starting position of the next task

## What Success Looks Like

The learner can preserve useful context or artifacts in a way that makes later work faster, better, or more reliable because the system can retrieve and apply that prior work.

## Good Lesson Shapes

- Memory system projects
- Knowledge curation lessons
- Artifact reuse case studies
- Continuity-focused workflow builds

## Avoid

- Storing everything
- Trusting memory without validation
- Treating archives as leverage if nobody can retrieve or use them

## Track-Specific Failure Mode

Accumulating context sludge. The learner saves everything — notes, outputs, prior conversations, artifacts — without curation, creating a growing pile of context that degrades retrieval quality and makes the system slower instead of smarter.

## Recommended Format Emphasis

| Format | Priority |
|--------|----------|
| `project` | Primary — memory and continuity systems are built |
| `case-study` | High — artifact reuse and retrieval quality examples |
| `reference` | Regular — curation checklists and memory hygiene guides |
| `concept` | Occasional — retrieval and trust principles |
| `lab` | Occasional — focused curation or retrieval tasks |
