# Track Calibration: Directing

## Core Learner Shift

Move from relying on the system to infer the goal into explicitly defining success, constraints, output shape, and verification.

## What Lessons Should Optimize For

- Clear briefs
- Acceptance criteria
- Verification thinking
- Better judgment about task framing
- Deliberate tradeoff setting

## Authoring Direction

- Make lesson outputs testable
- Show weak framing versus strong framing
- Use drills and comparisons generously
- Include explicit examples of success criteria, failure criteria, and checks
- Treat prompt writing as thinking work, not wording work

## What Success Looks Like

The learner can give one agent a precise brief that includes what "done" means and how the result should be checked.

## Good Lesson Shapes

- Brief-writing lessons
- Acceptance-criteria drills
- Specification labs
- Project scoping lessons
- Review and verification case studies

## Avoid

- Equating verbosity with precision
- Hiding verification in the background
- Drifting into multi-agent workflows unless the lesson is transitioning intentionally toward Delegating

## Track-Specific Failure Mode

Writing long prompts without defining success. The learner writes detailed instructions but never specifies what "done" looks like or how to check the result, producing verbose prompts that still fail to direct.

## Recommended Format Emphasis

| Format | Priority |
|--------|----------|
| `concept` | Primary — framing, verification, and judgment rules |
| `drill` | High — comparison and practice build framing skill fast |
| `lab` | Regular — specification and verification tasks |
| `case-study` | Regular — review and verification examples |
| `project` | Occasional — scoping exercises |
