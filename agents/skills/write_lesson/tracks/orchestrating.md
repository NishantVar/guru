# Track Calibration: Orchestrating

## Core Learner Shift

Move from ad hoc delegation into running a coordinated team of agents with stable roles, sequencing, and synthesis logic.

## What Lessons Should Optimize For

- Role design
- Dependency management
- Parallel work with explicit coordination
- Arbitration and synthesis
- Deciding when coordination cost is worth paying

## Authoring Direction

- Teach an operating model, not just a trick
- Give role definitions names and responsibilities
- Show how information moves between roles
- Make coordination logic visible
- Use architecture sketches, flow maps, or operating tables where helpful

## What Success Looks Like

The learner can run a repeatable multi-agent pattern with recognizable roles and deliberate coordination, rather than improvising the entire process each time.

## Good Lesson Shapes

- Role-system concepts
- Coordination case studies
- Multi-agent project builds
- Operating model templates

## Avoid

- Adding more agents just to look advanced
- Unclear arbitration rules
- Treating synthesis as an afterthought
- Teaching reusable infrastructure here if the real topic is Systemizing

## Track-Specific Failure Mode

Coordination overhead swallowing the gains. The learner creates elaborate multi-agent workflows where the cost of managing roles, dependencies, and synthesis exceeds the value the coordination was supposed to produce.

## Recommended Format Emphasis

| Format | Priority |
|--------|----------|
| `project` | Primary — stable coordination patterns are built, not described |
| `case-study` | High — coordination examples teach judgment effectively |
| `concept` | Regular — role design and coordination principles |
| `lab` | Occasional — focused coordination exercises |
| `drill` | Rare — orchestration is too holistic for isolated drills |
