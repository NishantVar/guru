# Track Calibration: Evolving

## Core Learner Shift

Move from a system that compounds into a system with an outer loop that evaluates and improves itself over time.

## What Lessons Should Optimize For

- Evaluation loops
- Workflow redesign
- Prompt and role revision
- Memory quality improvement
- System-level learning

## Authoring Direction

- Teach how to inspect the system itself
- Make metrics, criteria, and review cadences explicit
- Show how to decide what to change and why
- Include before-and-after comparisons based on evidence
- Keep the focus on real improvement, not recursive complexity

## What Success Looks Like

The learner can observe where an agentic system is drifting or underperforming, evaluate it deliberately, and improve some part of the workflow, memory, prompt stack, or role design based on evidence.

## Good Lesson Shapes

- Evaluation-framework concepts
- Redesign case studies
- Continuous-improvement projects
- Meta-workflow audits

## Avoid

- Complexity theater
- Vague "self-improving system" language
- Making meta-layers without measurement

## Track-Specific Failure Mode

Adding meta-complexity without strong evaluation. The learner adds improvement loops, review cadences, and meta-layers to their system without clear metrics or evidence that the changes actually improve outcomes, creating recursive overhead that looks sophisticated but doesn't help.

## Recommended Format Emphasis

| Format | Priority |
|--------|----------|
| `case-study` | Primary — evaluation and redesign are best taught through evidence |
| `project` | High — continuous improvement systems are built |
| `concept` | Regular — evaluation and improvement principles |
| `lab` | Occasional — focused audit or evaluation tasks |
| `drill` | Rare — meta-evaluation is too holistic for isolated drills |
