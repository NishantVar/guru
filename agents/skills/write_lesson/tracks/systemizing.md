# Track Calibration: Systemizing

## Core Learner Shift

Move from a human-managed operating model into reusable infrastructure that makes good agentic work repeatable.

## What Lessons Should Optimize For

- Reuse
- Standardization
- Reliability
- Verification paths
- Encoded workflow quality

## Authoring Direction

- Focus on artifacts that survive beyond one project
- Teach how to turn craft into defaults
- Include playbooks, harnesses, templates, checks, or structured loading patterns
- Make it clear what becomes reusable and by whom
- Emphasize outcome improvement over process sophistication

## What Success Looks Like

The learner can create or adopt a reusable workflow component that reliably improves how agent work gets done across tasks.

## Good Lesson Shapes

- Harness-building projects
- Reusable workflow labs
- Prompt and playbook design lessons
- Evaluation and review loop lessons
- System template case studies

## Avoid

- Elaborate process for its own sake
- One-off cleverness disguised as infrastructure
- Teaching persistence or memory as if that alone creates leverage

## Track-Specific Failure Mode

Overengineering process without better outcomes. The learner builds elaborate reusable systems, templates, and harnesses that look sophisticated but don't measurably improve the quality or speed of agent work.

## Recommended Format Emphasis

| Format | Priority |
|--------|----------|
| `project` | Primary — reusable infrastructure is built, not theorized |
| `reference` | High — playbooks, templates, and pattern libraries |
| `case-study` | Regular — before/after system comparisons |
| `concept` | Occasional — standardization and reuse principles |
| `lab` | Occasional — focused infrastructure tasks |
