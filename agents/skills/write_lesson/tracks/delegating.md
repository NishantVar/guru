# Track Calibration: Delegating

## Core Learner Shift

Move from one precise thread into splitting work across bounded agents or runs with clear ownership and manual synthesis.

## What Lessons Should Optimize For

- Task decomposition
- Ownership boundaries
- Handoff quality
- Separation of exploration, implementation, and review
- Recombining outputs without confusion

## Authoring Direction

- Teach the learner how to split work cleanly
- Make ownership explicit in examples
- Show what should stay local versus what should be delegated
- Include review steps after delegated work returns
- Keep the coordination light enough for one person to manage manually

## What Success Looks Like

The learner can split a task into bounded pieces, assign them to agents or runs intentionally, and synthesize the outputs back into one result.

## Good Lesson Shapes

- Decomposition labs
- Handoff templates
- Multi-run projects with bounded roles
- Compare-good-vs-bad delegation case studies

## Avoid

- Fuzzy roles
- Duplicated work
- Agent spawning as a status move
- Assuming a stable operating model that has not been taught yet

## Track-Specific Failure Mode

Spawning extra agents without boundaries. The learner creates multiple agent runs without clear ownership, leading to duplicated work, conflicting outputs, and synthesis that takes longer than doing the task in one thread.

## Recommended Format Emphasis

| Format | Priority |
|--------|----------|
| `lab` | Primary — bounded decomposition and handoff tasks |
| `project` | High — multi-run builds with clear ownership |
| `case-study` | Regular — good vs bad delegation comparisons |
| `concept` | Occasional — decomposition principles |
| `drill` | Occasional — handoff quality practice |
